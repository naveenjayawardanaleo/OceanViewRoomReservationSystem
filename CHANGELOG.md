# Changelog
All notable changes to the OceanView Resort Reservation System are documented in this file.

Version 1.0 – Final Project Submission
Date: March 2026

Added
- Complete OceanView Resort Reservation System
- User authentication using LoginServlet
- Room management (Add, Edit, Delete, View)
- Reservation creation and status tracking (Pending / Completed)
- Billing system with automated calculations
- VAT (18%) and Service Charge (10%) calculations
- Invoice generation with PAID watermark for completed reservations

Improved
- Login validation logic enhanced
- Contact number validation added
- User interface improvements for reservation form
- Room dropdown sorting by price for better usability

Documentation
- Added system usage documentation
- Added project README with setup instructions
- Added UI screenshots for documentation
- Added UML design diagrams
- Added testing documentation and test cases

Version 0.9 – Feature Completion
Date: February 2026

Added
- Reservation management functionality
- Bill generation feature
- Reservation completion workflow
- Database schema creation script

Improved
- Input validation for contact number field
- Error handling improvements

Version 0.8 – Initial Development
Date: February 2026

Added
- Initial project setup using Maven
- Database connection using DBConnection Singleton
- RoomsDAO and ReservationDAO implementation
- Basic project structure following MVC architecture
